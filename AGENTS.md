# stock(intra-day trading)/crypto 量化交易策略專案

## 核心架構組件

### 1. 後端伺服器
- 執行交易策略的即時運算和訊號生成
- 發送通知到目標節點（交易帳戶API或Discord等通訊管道）
- 提供API接口供前端和外部系統調用

### 2. 前端Dashboard
- 回測結果視覺化（統計數據、績效指標）
- K線(OHLC)或折線圖(line chart)顯示交易進出場的時間和價格
- 策略對比和性能監控界面

### 3. 數據回測引擎
- 可使用自建框架或既有工具（如backtrader）
- 完整的績效評估和風險分析

### 4. 策略存放目錄
- 使用統一的策略抽象class作為基底
- 所有策略實現放置於`/strategies`目錄
- 標準化的策略接口和執行流程

## 品質標準

### 策略績效要求
- **評估函數**: 要為每個策略設計評估函數，以年化報酬為基底，透過夏普比率、最大回撤或者其他高階數據來進行調整，注意：當你設計評估函數，背後需要數學理論支持

### 技術架構原則
- **機器學習驅動（可選，也可以單純用網格搜索找好用的技術指標組合，但還是要評估這個指標的參數在不同時間的泛化度，因為BTC這類資產的特性隨時間和政府規則的變動遷移較明顯）**: 以XGBoost等ML模型為核心
- **並行處理**: 計算密集任務支援多核心/GPU加速
- **模組化設計**: 組件職責單一，易於維護和擴展
### 架構風格備忘
- **資料處理/訊號產生**：偏向函數式設計，強調可組合性與單元測試便利性。
- **風控/執行/排程**：當涉及長期狀態管理時，可在外層採輕量 OOP 容器，內部仍以純函數描述邏輯，平衡維護成本與擴充性。


## 開發規範

### 程式碼風格
- **註解語言**: 繁體中文
- **日誌語言**: 英文
- **日誌等級**: INFO用於重要資訊，DEBUG用於調試
- **輸出結果**: 評估如何快速瀏覽該次訓練或回測效果 or 完整存到 csv/SQL-like/NoSQL/只靠logging

## 最終目標

### 核心功能
- 存放策略 -> 策略回測 -> 即時交易訊號生成 -> 後續執行
- 即時訊號產生後，務必確保回測時該指標真的也會出現，若兩者出現差異則可能白作工

## 中文檔案讀取（UTF‑8）
- 在本專案以 Codex CLI/PowerShell 檢視含中文的文字檔，請強制以 UTF‑8 解碼並輸出，避免亂碼。
  - 範例：
    - `powershell -File scripts/cat_utf8.ps1 -Path README.md -Head 60`
    - `powershell -File scripts/cat_utf8.ps1 -Path strategies/stats_based_v0/feature_engineer.py`
- 若臨時需要直接以 PowerShell 讀檔，請同一行設定輸出與編碼：
  - `[Console]::OutputEncoding = New-Object System.Text.UTF8Encoding($false); Get-Content -LiteralPath <file> -Encoding UTF8 -TotalCount 60`
- 內容搜尋仍建議用 `rg`；需要顯示檔案內容時，再以上述 UTF‑8 方法開檔。
